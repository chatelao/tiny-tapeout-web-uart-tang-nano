# WASM Integration for Tiny Tapeout Simulation

This project supports high-fidelity, stateful simulation of Tiny Tapeout (TT) modules using WebAssembly (WASM). This allows for a "Digital Twin" of the FPGA logic to run directly in the browser.

## Architecture

The simulation integration consists of three main components:
1.  **Digital Twin (WASM):** A compiled C/C++ or Verilog-to-C++ (via Verilator) module that implements the TT design logic.
2.  **Emscripten Wrapper (`web/digital_twin_Full.js`):** The glue code generated by Emscripten that exposes the WASM functions to JavaScript.
3.  **Web Application (`web/app.js`):** The main application logic that manages the simulation state, handles UI events, and interacts with the WASM module.

## The `DigitalTwin` Class

The WASM module exposes a `DigitalTwin` class (via Emscripten's `embind`) with the following interface:

### Input Methods

*   **`set_ui_in(uint8_t value)`**: Sets the state of the 8-bit `ui_in` input bus.
*   **`set_uio_in(uint8_t value)`**: Sets the state of the 8-bit `uio_in` input/output bus (when used as input).
*   **`set_ena(bool value)`**: Sets the state of the `ena` (enable) signal.
*   **`set_rst_n(bool value)`**: Sets the state of the `rst_n` (active-low reset) signal.

### Control Methods

*   **`step()`**: Advances the simulation by one clock cycle. In the web tester, this is typically called on a logical rising edge of the `clk` signal.

### Output Methods

*   **`get_uo_out()`**: Returns the current 8-bit value of the `uo_out` output bus.
*   **`get_uio_out()`**: Returns the current 8-bit value of the `uio_out` input/output bus (when used as output).
*   **`get_uio_oe()`**: Returns the current 8-bit value of the `uio_oe` (output enable) bus, indicating which `uio` pins are currently configured as outputs.

## Integration in `web/app.js`

The web application initializes the WASM module and uses it when the "Use Full FP8 WASM" toggle is enabled.

```javascript
let digitalTwin = null;
let wasmReady = false;

// Initialize the WASM module
if (typeof Module !== 'undefined') {
    Module['onRuntimeInitialized'] = () => {
        digitalTwin = new Module.DigitalTwin();
        wasmReady = true;
    };
}

function mockBoard(uiValue, uioInValue, clkVal, rstVal, enaVal) {
    if (useFullWasm.checked && wasmReady && digitalTwin) {
        // Update inputs
        digitalTwin.set_ui_in(uiValue);
        digitalTwin.set_uio_in(uioInValue);
        digitalTwin.set_ena(enaVal === 1);
        digitalTwin.set_rst_n(rstVal === 1);

        // Advance simulation on rising edge
        if (clkVal === 1) {
            digitalTwin.step();
        }

        // Return outputs
        return {
            uo_out: digitalTwin.get_uo_out(),
            uio_out: digitalTwin.get_uio_out(),
            uio_oe: digitalTwin.get_uio_oe()
        };
    } else {
        // Fallback to simple XOR emulation
        const result = (uiValue ^ uioInValue) & 0xFF;
        return {
            uo_out: result,
            uio_out: 0,
            uio_oe: 0
        };
    }
}
```

## Example: FP8 Multiplier Simulation

One specific use case implemented in this project is the simulation of an 8-bit Floating Point (FP8) multiplier. When the WASM simulation is active, it performs stateful multiplication based on the TT design, providing a high-fidelity representation of how the hardware would behave on the Tang Nano 4K.
